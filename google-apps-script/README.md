# Google Apps Script（集計 API）

スプレッドシートに紐づけた GAS で、**指定セッションの集計結果を JSON** で返します。ダミーデータを入れたあとの「次の一手」として利用してください。

## 手順

1. **スプレッドシートを開く**（社員マスター・セッション・レスポンスの3タブがあるブック）。
2. **拡張機能 → Apps Script** でエディタを開く。
3. デフォルトの `Code.gs` の中身をすべて削除し、このフォルダの **`Code.gs` の内容をコピーして貼り付け**る。
4. **保存**（ディスクアイコン）し、左上の「実行」で `doGet` を一度選んで実行すると、初回だけ **権限承認**（スプレッドシートの参照）を求められる。承認する。
5. **デプロイ → 新しいデプロイ** → 種類「ウェブアプリ」  
   - 次のユーザーとして実行: **自分**  
   - アクセスできるユーザー: **全員**（課題・検証用。本番は要検討）  
   - デプロイして **ウェブアプリの URL** を控える。
6. ブラウザで次を開く（`202605` など、セッションシートの ID 値に置き換え）:

   `https://script.google.com/macros/s/.../exec?sessionId=202605`  
   クエリ名は **`sessionId`（推奨）** 。`sessionID` や `session_id` でも可（スクリプトが吸収）。

   セッション列名が **`session_ID`** の場合も対応済み（`Code.gs` の `sessionIdFromRow_`）。

7. **JSON が表示されれば成功**。

## 次の段階: 集計ダッシュボード（HTML）

リポジトリの **`admin-ui/dashboard.html`** をブラウザで開き、次を行う。

1. プロジェクトのルートでターミナルを開き、例: **`npx --yes serve .`** または **`npx --yes serve admin-ui`** でローカル HTTP サーバーを起動する（`file://` 直開きだと `fetch` が CORS で失敗することがある）。
2. ブラウザで `http://localhost:3000/admin-ui/dashboard.html`（ポートは `serve` の表示に合わせる）を開く。
3. **ウェブアプリ URL** に `https://script.google.com/macros/s/.../exec` を貼り、**セッション ID** に `202605` などを入力し、「**集計を読み込む**」を押す。
4. よければ「**URL をブラウザに保存**」で次回の入力を省略。

## シート名・ヘッダー

`Code.gs` 上部の定数は、ブックのタブ名と **1行目のヘッダー文字列**と一致させる。

| 定数 | 想定タブ名 |
|------|------------|
| `SHEET_EMPLOYEES` | 社員マスター |
| `SHEET_SESSIONS` | セッション |
| `SHEET_RESPONSES` | レスポンス |

**レスポンスの `ステータス` 列**には README の保存値（`safe` / `minor_injury` / `need_help` / `other`）を入れること。日本語ラベルだけだと集計が `unknown` になる。

**在籍フラグ**（社員マスター）が `はい` / `1` / `TRUE` / `在籍` のいずれかなら対象者に含める。列がない・空なら全員を対象とする。

## 返却 JSON の形（概要）

`session`, `totals`（対象・回答済・未回答・要救護）, `byStatus`, `needHelpList`, `noResponseList`, `byDepartment`（部署別の回答済／未回答件数）を含む。詳細は `Code.gs` 内の `buildPayload_` を参照。
